# 学习笔记整理助手

把你看过的内容（YouTube 视频、文章、文字）整理成一份小白友好的中文学习笔记，带术语解释，可以存档也可以直接分享给观众。

---

## 【要素六】依赖环境

运行前需要准备：

**必须有：**
- Node.js（v18 以上版本）

**处理 YouTube 视频时需要：**
- Supadata API key（免费注册：https://supadata.ai）
- 设置方式：`export SUPADATA_API_KEY=你的key`

**保存到本地文件夹时需要：**
- 在要素五的预设配置里填好你的本地保存路径
- 路径示例：`~/Documents/学习笔记/`（Mac）或 `C:\Users\你的名字\Documents\学习笔记\`（Windows）

**保存到飞书时需要：**
- 飞书 Tenant Access Token（用于授权写入飞书知识库）
- 获取方式：登录飞书开放平台 https://open.feishu.cn → 创建应用 → 开通 `wiki:wiki` 权限（应用身份）→ 用 App ID + App Secret 换取 Token
- 设置方式：在系统环境变量里新建 `FEISHU_USER_ACCESS_TOKEN`，值填 Token
- 飞书知识库页面的 wiki_node_token（从知识库链接里复制）
- 链接示例：`https://my.feishu.cn/wiki/YGaOwSKigiC6SJkjjEncfdSynfh` → wiki_node_token 就是 `YGaOwSKigiC6SJkjjEncfdSynfh`

## 【要素一】触发描述

以下情况触发这个 skill：

- 用户粘贴了 YouTube 链接（包含 youtube.com 或 youtu.be）
- 用户粘贴了文章/博客链接
- 用户粘贴了一段文字内容
- 用户说了这些话：「帮我整理」「做成笔记」「我看了这个」「帮我总结」「我不太懂」「存到飞书」

---

## 【要素三】执行步骤

### 第一步：判断输入类型，选择对应处理方式

**情况 A：YouTube 链接**
运行字幕抓取脚本：

```bash
cd ${SKILL_DIR}/scripts && node fetch-transcript.js "用户粘贴的链接"
```

脚本会输出视频字幕文字。如果脚本报错（比如没有 API key、视频无字幕），告诉用户：
「这个视频暂时无法自动获取字幕，你可以手动把视频字幕或笔记粘贴给我，我来帮你整理。」

**情况 B：文章/博客链接**
用 web_fetch 抓取文章正文，提取主要内容。

**情况 C：直接粘贴的文字**
直接使用这段文字，不需要额外操作。

**情况 D：什么都没给**
问用户：「你想整理什么内容？可以把 YouTube 链接、文章链接，或文字直接粘贴给我。」

---

### 第二步：按格式生成笔记

输出语言：中文。语气：轻松，像朋友聊天。

---

**📌 一句话总结**
用一句话说清楚这个内容在讲什么，让完全不懂的人也能看懂。

**🧠 核心要点**
列出 3-5 个最重要的观点，每条 2-3 句话。只挑最有价值的，不要罗列所有内容。

**📖 术语解释**
把正文中的专业词汇挑出来，每个用一句大白话解释，最好附上生活类比。
格式：**词汇**：解释（类比）
如果没有专业词汇，省略这一块。

**💡 可以怎么用**
用第一人称写：如果把这个知识用在实际生活或工作中，能做什么？给一个具体例子。

**🔗 原始来源**
注明内容来源（链接或标题）。

---

### 第三步：询问并执行保存方式

笔记生成后问用户：「这份笔记你想怎么保存？」

**选项 1：直接看就好**
不需要任何操作，结束。

**选项 2：保存到本地文件夹**
运行本地保存脚本：
```bash
cd ${SKILL_DIR}/scripts && node save-local.js "笔记内容" "文件名"
```
文件会保存到要素五里设定的默认路径。保存成功后告诉用户：「已保存到 ~/Documents/学习笔记/文件名.md」

**选项 3：保存到飞书知识库**
先确认用户是否提供过飞书知识库链接。
- 如果没有：问用户「请把你的飞书知识库页面链接发给我」，从链接中提取 wiki_node_token（/wiki/ 后面那串字符）
- 如果有：直接使用已知的 wiki_node_token

然后运行飞书写入脚本：
```bash
cd ${SKILL_DIR}/scripts && node save-feishu.js "笔记内容" "wiki_node_token"
```
保存成功后告诉用户：「已追加到你的飞书知识库，可以去查看了。」

**选项 4：两个都要（本地 + 飞书）**
依次执行选项 2 和选项 3。

## 【要素五】预设配置

没有特别说明时，默认：

- 语言：中文输出
- 笔记长度：500 字以内
- 术语解释：开启
- 保存方式：先在对话里显示，再询问是否保存

**本地保存默认路径：**
`~/Documents/学习笔记/`
用户可以随时说「改成存到桌面」或任意路径，AI 按新路径保存。

文件命名规则：`YYYY-MM-DD_内容标题.md`
示例：`2026-06-10_TLDW视频工具介绍.md`

**飞书默认 wiki_node_token：**
首次使用时 AI 问用户：「请把你的飞书知识库页面链接发给我，我来提取 wiki_node_token，下次就不用再问了。」

---

## 【要素四】附带资源

`scripts/fetch-transcript.js` — 调用 Supadata API 抓取 YouTube 字幕。处理 YouTube 链接时先运行。

`scripts/save-local.js` — 把笔记保存为 .md 文件到本地指定文件夹。

`scripts/save-feishu.js` — 调用飞书 API，把笔记内容追加写入飞书文档。

---

## 【要素七】边界说明

以下情况不处理，告知用户原因：

- **私密视频或需要登录才能看的视频** — 无法获取字幕，请用户手动提供内容
- **没有字幕的视频** — 无法处理，建议用户手动整理后粘贴给我
- **超过 3 小时的视频** — 字幕太长，只处理前 1 小时内容，其余略过并告知用户
- **非文字内容（纯音乐、纯图片视频）** — 无法整理，请用户提供文字描述
- **不读取任何私人数据或需要登录才能访问的页面**
